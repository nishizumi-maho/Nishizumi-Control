# Read This First

**Dominant Control for iRacing** — Accessibility, Voice Control, and HUD Manager (v5.0.0)

## Quick Order for Best Results
1. Start Dominant Control **first** to avoid FFB conflicts.
   - Or enable **Keyboard Only Mode** if you only use keyboard bindings.
2. Start iRacing and join a **private practice session**, then click **Drive**.
3. In the app: switch to **CONFIG** → **Step 3: Scan driver controls** → bind keys → set HUD → save profile.
4. Switch to **RUNNING** mode and test.

## What’s New in 5.0
- New **Main/Options** layout with a guided step list for car/track, devices, and scanning.
- Voice & audio settings consolidated in **Options → Voice/Audio Settings**.
- Manual rescan and clear-target hotkeys plus expanded automation toggles.
- Optional **whisper.cpp** offline recognition using local GGML/GGUF models.

## Essential Settings
- **Device Management:** Keyboard Only Mode or selectively enable button boxes via **Manage Devices**.
- **Timing Profiles:** Aggressive / Casual / Relaxed / Custom (per profile).
- **HUD / Overlay:** choose variables, colors, opacity; drag to position.
- **⚡ Combos:** one key/button applies multi-control presets.
- **Voice Control:** set Vosk or whisper.cpp model paths, enable Voice Commands, test phrases.
- **Text-to-Speech:** optional spoken confirmations with selectable output device.

## File Locations
- Configs: `%APPDATA%\\DominantControl\\configs\\` (safe to back up).
- Application: extracted folder or repository clone.
